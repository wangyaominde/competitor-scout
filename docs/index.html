核心能力

从配置到扫描、匹配、可视化与定时盯盘，一条工作流走完。

01

智能扫描

Discover → Enrich → 威胁 → Agent 校验。指示灯 + 心跳日志，长请求不会像假死。

02

遍历匹配我方产品

对每个竞品，按我方产品逐条做 BM25+RAG，输出 threat_vs 明细并取最高威胁。

03

3D 威胁空间

多维能力坐标散点：我方基准 vs 竞品球体，拖拽旋转、缩放、点选详情。

04

击败路径

AI 生成分阶段 roadmap：差距、必做项、价格与渠道策略，并可视化目标点。

05

Loop 引擎

cron 定时扫描，高威胁桌面通知；后台与手动扫描共用历史与日志。

06

本地数据主权

密钥与竞品库仅存本机。公开仓库不含 `.data`；打包后使用系统 userData。

威胁流水线

自动判定负责打分，人工确认负责入库——职责清晰。

  1. 规则基线 价格、规格、渠道、定位等结构化特征分
  2. BM25 召回 从竞品语料检索邻域证据
  3. RAG 判定 证据 + 我方产品 + 目标竞品 → LLM 威胁分与理由
  4. 逐产品遍历 多产品时一条一条匹配，取最高并保留明细
  5. 人工筛选 待确认 → 确认入库 / 忽略

公开仓库，不公开你的密钥

GitHub 上的代码是极简源码树。API Key、竞品库、扫描历史都在本地 .data/（已 gitignore）或系统用户目录。

  • CI 防泄漏：check-secrets.js
  • 三系统自测：Ubuntu / macOS / Windows
  • Release：tag v* 构建 dmg / exe

快速开始

需要 Node.js 20+ 与可访问的 OpenAI 兼容接口（或 Ollama）。

终端
git clone https://github.com/wangyaominde/competitor-intel.git
cd competitor-intel
npm install
npm start

首次路径

  1. 配置 LLM（Base URL / Key / Model）
  2. 添加一个或多个「我方产品」
  3. 智能扫描（可开遍历匹配）
  4. 竞品库确认 · 查看 3D / 击败路径
